## 광고 차단 및 관련 기술의 이해

광고 차단을 적용하기 위한 규칙 모음인 필터(Filter list)는 대체로 .txt의 확장자를 가집니다.

### 네트워크(DNS) 필터링
브라우저가 서버에 자원(이미지, 스크립트, 광고 데이터 등)을 요청하기 전에 개입하여 해당 요청을 차단합니다.
*   **원리**: HTTP/HTTPS 요청 주소를 필터 리스트와 대조하여 일치하면 전송을 중단합니다.
*   **특징**: 광고 데이터를 아예 다운로드하지 않으므로 데이터 절약과 속도 향상에 가장 효과적입니다. 하지만 후술할 한계점인 도메인 주소가 주요 콘텐츠와 동일하거나 유동적으로 바꾸는 광고에는 작동하지 않습니다.

### 코스메틱(요소 숨김) 필터링
데이터 요청은 막지 못했거나 이미 로드된 페이지 내에서 광고가 차지하는 공간을 강제로 숨기는 방식입니다.
*   **원리**: CSS 선택자를 사용하여 광고 요소의 속성을 보이지 않도록 변경합니다.
*   **필요성**: 광고 서버가 웹사이트 본체와 같은 도메인을 사용하거나, 광고가 삭제된 자리에 남는 공백을 제거하기 위해 사용합니다. 일부 광고 차단기는 동적으로 생성되는 광고 요소를 실시간 감시하여 삭제하기도 합니다.

### 스크립트릿 및 인젝션
단순한 차단이나 숨기기로 해결되지 않는 복잡한 광고나 광고 차단 방지(Anti-Adblock) 스크립트를 무력화하기 위해 사용합니다.
*   **원리**: 웹사이트 내에 특정 자바스크립트 조각을 강제로 주입하여 광고 관련 함수를 빈 함수로 교체하거나, 차단기 감지 로직이 가짜 정보를 읽도록 속입니다.

## 2. 유저스크립트(사용자 스크립트, UserScript)의 이해

유저스크립트는 웹사이트의 동작 방식을 직접 수정하거나 새로운 기능을 추가하는 독립적인 자바스크립트 파일입니다. 확장자 .user.js를 일반적으로 가지고 있습니다.

### 2.1 스크립트 매니저가 필요한 이유
브라우저는 보안상 외부 스크립트가 웹사이트의 내부 데이터에 무분별하게 접근하는 것을 차단합니다. 따라서 유저스크립트를 안전하게 실행하고 관리하기 위해 '매니저(Tampermonkey 등)'라는 중간 매개체가 필요합니다.
*   **샌드박스**: 스크립트가 페이지의 원래 코드와 충돌하지 않도록 격리된 환경에서 실행됩니다.
*   **권한 관리**: `@grant` 구문을 통해 스크립트가 브라우저의 특정 API(저장소 접근, 크로스 도메인 요청 등)를 사용할 수 있도록 허용합니다.

### 2.2 메타데이터와 실행 시점
유저스크립트 상단에는 `// ==UserScript==`로 시작하는 설정 값이 포함됩니다.
*   **@match / @include**: 스크립트가 작동할 사이트 주소를 지정합니다.
*   **@run-at**: 페이지 로딩 중 어느 시점에 코드를 실행할지 결정합니다. (예: DOM 생성 직후, 전체 로딩 완료 후 등)

### 데스크톱 브라우저 환경 (Windows, macOS, Linux)

**[광고 차단 필터 도구]**
*   **uBlock Origin (uBO)**: [권장] 현존하는 가장 가볍고 강력한 차단기입니다. MV2 환경에서 완벽한 성능을 내며, 복잡한 스크립트릿 처리와 CNAME 언클로킹(Uncloaking) 기능을 지원합니다.
    *   *설치법*: `대시보드(설정)` > `필터 목록` > 하단 `가져오기...` 체크 후 본 리포지토리의 `filter.txt` URL 적용.
*   **AdGuard 브라우저 확장**: uBO와 유사한 성능을 내지만, 메모리 점유율이 약간 더 높습니다. AdGuard 고유 필터 문법을 가장 완벽하게 지원합니다.
*   *주의사항*: uBO와 AdGuard 확장을 동시에 켜두면 충돌하여 페이지 로드가 지연되므로 반드시 하나만 사용하세요.

**[유저스크립트 매니저]**
*   **Tampermonkey**: [권장] 가장 널리 쓰이는 독점 소프트웨어. 기능이 가장 많고 호환성이 우수합니다.
*   **Violentmonkey**: 오픈소스이며 Tampermonkey보다 훨씬 가볍고 빠릅니다. 최신 브라우저 환경에 최적화되어 있습니다.
*   **Greasemonkey**: Firefox 환경의 원조 매니저지만, 현재는 호환성 문제로 권장하지 않습니다.
    *   *설치법*: 매니저 확장 프로그램 설치 후, 본 리포지토리의 `.user.js` URL을 클릭하면 자동으로 설치 프롬프트가 호출됩니다.

### Windows
브라우저 확장이 아닌 데스크톱 애플리케이션 형태의 유료/무료 프로그램입니다.

*   **작동 원리**: 브라우저 레벨이 아닌, OS의 네트워크 스택(WFP 등)에서 로컬 프록시를 생성해 패킷을 가로챕니다. 브라우저뿐만 아니라 스팀, 카카오톡 등 PC 내 모든 프로그램의 광고를 차단합니다.
*   **HTTPS 필터링**: 암호화된 트래픽을 열어보기 위해 중간자 공격(MitM) 방식을 사용합니다. AdGuard의 자체 로컬 루트 인증서를 OS에 강제 설치하여 HTTPS 패킷을 복호화하고 필터링한 뒤 다시 암호화해 브라우저로 보냅니다.
*   *설치법*: `설정` > `광고 차단기` > `필터 추가`에서 `filter.txt` 등록. `설정` > `확장 프로그램`에서 유저스크립트(`.user.js`) 등록 가능 (별도의 Tampermonkey 필요 없음).
